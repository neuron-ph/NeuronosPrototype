# Booking Information Screen Spec

Purpose: define exactly what the booking information screen should show for each Neuron service booking, including section order, field labels, field keys, input controls, required rules, auto-fill behavior, and conditional visibility.

Use this as the implementation checklist for booking create/edit/detail screens. The broader schema rationale lives in `src/docs/BOOKING_SCHEMA_STANDARD.md`.

## Screen Rules

Every booking screen must use this order:

| Order | Screen section | Meaning |
|---|---|---|
| 1 | General Information | Shared booking identity, ownership, assignment, status, and source lineage. |
| 2 | General Specific Booking Information | Service-specific "general" fields that determine what details are shown. |
| 3 | Booking Information Details | Operational execution details for the chosen service, movement, mode, and status. |

Every field row below is written as an implementation contract:

| Column | Meaning |
|---|---|
| Field key | Canonical key for form state and new `details` JSON keys. Use snake_case for new work. |
| Label | Exact screen label. |
| Control | UI control to render. |
| Required | `Yes`, `No`, or `Conditional`. |
| Show when | Visibility rule. `Always` means show for that service whenever the screen is open. |
| Source / behavior | Auto-fill, options, profile source, validation, or implementation notes. |

## Shared General Information

Render these fields for every service before the service-specific sections.

| Field key | Label | Control | Required | Show when | Source / behavior |
|---|---|---|---|---|---|
| `booking_name` | Booking Name | Free text | Yes | Always | User-entered operational label. Store in `bookings.name`. |
| `booking_number` | Booking Number | Auto-fill read-only | Yes | Edit/detail after number exists | Generated by service prefix. Store in `bookings.booking_number`. |
| `service_type` | Service Type | Auto-fill read-only | Yes | Always | Selected module. Store in `bookings.service_type`. |
| `customer_name` | Customer Name | Searchable dropdown | Yes | Always | Source from customer profiles. Store in `bookings.customer_name`. |
| `account_owner` | Account Owner | Auto-fill with searchable override | Yes | Always | Default from source quotation/project/customer profile/current user. |
| `account_handler` | Account Handler | Auto-fill with searchable override | Yes | Always | Default from customer/service preference or assigned handler. |
| `services` | Service/s | Multi-select | Yes | Always | Options from service catalog filtered by `service_type`. |
| `quotation_reference_number` | Quotation Reference Number | Auto-fill read-only with manual override option | Conditional | Always except Trucking may display as hidden lineage | Required when created from quotation/contract. For Trucking, display `container_number` as the visible operational reference if client preference is confirmed. |
| `project_number` | Project / Contract Number | Auto-fill read-only | Conditional | When linked to project or contract | Store canonical link in `bookings.project_id` and `bookings.contract_id`; display project/contract number for humans. |
| `movement_type` | Movement | Segmented control | Conditional | Brokerage, Forwarding, Trucking, Marine Insurance | Options: Import, Export. Add Domestic only for Trucking if business confirms domestic trucking. Store in `bookings.movement_type`. |
| `status` | Status | Dropdown | Yes | Always | Options must be service-specific. Store in `bookings.status`. |
| `team_assignment` | Team Assignment | Auto-fill assignment block with searchable overrides | Recommended | Always | Manager, supervisor, handler from customer/service defaults. Store top-level booking assignment IDs/names. |

### Shared Status Options

| Service | Status dropdown options |
|---|---|
| Brokerage | Draft, Waiting for Arrival, Ongoing, Delivered, Billed, Paid, Audited, Cancelled |
| Forwarding | Draft, Ongoing, In Transit, Delivered, Completed, Billed, Paid, Cancelled |
| Trucking | Draft, Ongoing, Delivered, Empty Return, Liquidated, Billed, Paid, Cancelled |
| Marine Insurance | Draft, Ongoing, Issued, Billed, Paid, Cancelled |
| Others | Draft, Ongoing, Completed, Billed, Paid, Cancelled |

## Data Entry Defaults

| Field family | Default behavior |
|---|---|
| Customer, consignee, shipper | Search existing profile first. Allow combo-box creation only if master-data workflow allows it. |
| Account owner, account handler, team assignment | Auto-fill from quotation/project/customer preference/current user. Allow override for authorized roles. |
| Quotation reference, project, contract | Auto-fill from source record. Keep read-only unless user has explicit unlink/relink flow. |
| Ports, airports, countries | Use searchable dropdowns from managed lists. |
| Carrier, agent, consolidator, insurer, shipping line | Use searchable dropdowns from profile tables; allow combo-box fallback during onboarding. |
| Dates | Use date input when time does not matter. Use date-time input when dispatch timing matters. |
| Monetary fields | Use currency input with currency code support if multi-currency is in scope. |
| Weights | Use number input with unit label. Default kg. |
| Dimensions | Use structured dimensions if possible; use free text only for initial rollout. |
