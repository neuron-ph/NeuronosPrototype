Critical Architectural Patterns

Module Router Pattern
Each major module (Accounting, Pricing, BD, Operations) has a top-level router component accepting a view prop, delegating to sub-pages. Never bypass this shell.
Unified Tab Reuse
Financial tabs are never recreated. Always import UnifiedBillingsTab, UnifiedInvoicesTab, UnifiedCollectionsTab, UnifiedExpensesTab with props.
Portal Dropdown
CustomDropdown uses createPortal → document.body, position:fixed, zIndex:9999, scroll-reposition (not close). Never change this.
RouteGuard + dept/role
All protected routes wrapped by RouteGuard checking allowedDepartments & requireMinRole. App Mode System: "Essentials" vs "Full Suite".
JSONB details columns
quotations, projects, bookings, evouchers store overflow fields in a details JSONB column. Always merge: {...data?.details, ...data}
quotations dual-type
Same quotations table serves both. Distinguished by quotation_type: "spot" | "contract". ContractDetailView and QuotationDetail both read from it.
DataTable standard
All data tables use /components/common/DataTable.tsx. Never build custom tables from scratch.
Blueprint-Driven Dev
45 blueprint files in /src/docs/blueprints/. Read blueprint → create/update blueprint → wait for Go Ahead → implement phase by phase.
