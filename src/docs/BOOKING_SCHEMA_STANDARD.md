# Booking Schema Standard

Purpose: standardize Neuron service bookings into a consistent three-part structure while preserving the client-provided inputs for Brokerage, Forwarding, Trucking, Marine Insurance, and Others.

For the exact field-by-field booking screen build checklist, including input controls and conditional display rules, see `src/docs/BOOKING_INFORMATION_SCREEN_SPEC.md`.

## Target Structure

Every service booking should be displayed and stored using the same conceptual structure:

| Section | Meaning | Examples |
|---|---|---|
| General Information | Fields genuinely repeated across services and useful for searching, assignment, status, and lineage. | Customer, account owner, account handler, service type, quotation reference, status |
| General Specific Booking Information | Fields that are currently presented as "general" by the client, but are actually specific to one service's workflow. | Brokerage mode, forwarding incoterm, trucking truck type |
| Booking Information Details | The real service execution data: shipment, delivery, policy, cargo, customs, and operational details. | MBL, HBL, container numbers, driver, BL/AWB, amount insured |

The database already supports this pattern:

| Storage area | Use |
|---|---|
| `bookings` top-level columns | Universal fields such as `service_type`, `booking_number`, `customer_name`, `status`, `movement_type`, `mode`, `project_id`, `contract_id`, `manager_id`, `supervisor_id`, `handler_id` |
| `bookings.details` JSONB | Service-specific fields from the General Specific and Booking Information Details sections |

## Input Control Types

Use these input types consistently across booking forms:

| Input type | Use when | Notes |
|---|---|---|
| Auto-fill read-only | Value is derived from quotation, project, contract, current user, customer profile, or team defaults. | Allow manual override only if the operational team has a real exception workflow. |
| Searchable dropdown | Value should come from a managed profile table but users may need search. | Customer, consignee, shipper, carrier, agent, consolidator, ports, warehouse, insurer. |
| Combo box | Prefer profile value, but allow new manual entry when profile is incomplete. | Useful during onboarding while master data is still being built. |
| Dropdown | Closed list with known options. | Mode, cargo type, cargo nature, customs entry, status, yes/no fields. |
| Multi-select | More than one controlled option can apply. | Services, sub-services, examination types. |
| Segmented control | Small mutually exclusive mode switch. | Import/export movement, brokerage type if treated as primary selector. |
| Date input | Calendar date only. | ETD, ETA, ETB, LCT, date delivered, date issued. |
| Date-time input | Operational date plus time. | Tagging time, cut-off time, pull-out time if time is required. |
| Free text | Short unstructured value. | MBL, HBL, registry number, flight number. |
| Textarea | Longer notes or instructions. | Delivery instructions, remarks, additional information. |
| Number input | Quantity, weight, days, package count. | Use unit label or paired unit selector. |
| Currency input | Money values. | Duties and taxes paid, brokerage fee, shipping charges. |
| Boolean toggle/dropdown | Yes/no value. | Container deposit, stackable, CY fee, early gate in. |
| Multi-value input | Same field can contain multiple values. | Container numbers, seal numbers, HS codes. |
| Repeater table | Repeating structured rows. | Containers, examinations, trucking destinations, charges. |

## Universal General Information

These fields should appear for every booking type unless explicitly marked system-only.

| Canonical field | Label | Input type | Required | Source / behavior | Storage |
|---|---|---|---|---|---|
| `booking_name` | Booking Name | Free text | Yes | User-entered operational label. | `bookings.name` |
| `booking_number` | Booking Number | Auto-fill read-only | Yes | Generated by service prefix. | `bookings.booking_number` |
| `service_type` | Service Type | Auto-fill read-only | Yes | Selected service module: Forwarding, Brokerage, Trucking, Marine Insurance, Others. | `bookings.service_type` |
| `customer_name` | Customer Name | Searchable dropdown | Yes | Customer profile. | `bookings.customer_name` |
| `account_owner` | Account Owner | Auto-fill or searchable dropdown | Yes | Prefer quotation/project owner or customer default. | `details.account_owner` until promoted |
| `account_handler` | Account Handler | Auto-fill or searchable dropdown | Yes | Prefer customer default or assigned handler. | `details.account_handler` until promoted |
| `services` | Service/s | Multi-select | Yes | Service catalog items available for the chosen service type. | `details.services` |
| `quotation_reference_number` | Quotation Reference Number | Auto-fill read-only, with optional manual entry | Conditional | Auto-fill from source quotation/contract. Required when booking was created from pricing. | `details.quotation_reference_number` |
| `project_number` | Project / Contract Number | Auto-fill read-only | Conditional | Auto-fill when linked to project or contract. | `bookings.project_id`, `bookings.contract_id`, `details.project_number` |
| `status` | Status | Dropdown | Yes | Service-aware status profile. | `bookings.status` |
| `movement_type` | Movement | Segmented control | Conditional | Import/export where operationally relevant. Current Neuron supports this on all services; keep available for lineage. | `bookings.movement_type` |
| `team_assignment` | Team Assignment | Auto-fill with searchable override | Recommended | Manager, supervisor, handler from customer/service defaults. | `bookings.manager_id`, `bookings.supervisor_id`, `bookings.handler_id` |

### Status Standard

Current Neuron uses a general lifecycle (`Draft`, `Created`, `Confirmed`, `In Transit`, `Delivered`, `Completed`, `Cancelled`). Client-provided operations statuses are more service-specific. The recommended standard is:

| Service | Status options |
|---|---|
| Brokerage | Draft, Waiting for Arrival, Ongoing, Delivered, Billed, Paid, Audited, Cancelled |
| Forwarding | Draft, Ongoing, In Transit, Delivered, Completed, Billed, Paid, Cancelled |
| Trucking | Draft, Ongoing, Delivered, Empty Return, Liquidated, Billed, Paid, Cancelled |
| Marine Insurance | Draft, Ongoing, Issued, Billed, Paid, Cancelled |
| Others | Draft, Ongoing, Completed, Billed, Paid, Cancelled |
